# REDBIRD Interactive Book v0.9 — dashboard architecture

## Purpose

v0.8 proved the visual direction: late-1980s adventure-game dithering, REDBIRD book interface, and a working Netlify deployment.

v0.9 shifts the project from a simple mockup into a more useful project interface.

The goal is not to replace Notion. The goal is to let Notion become the structured source-of-truth and let the Netlify site become a polished, readable interface layer.

## Core architecture

```text
Notion dashboards → exported JSON snapshot → GitHub repo → Netlify interface
```

## Roles

| Layer | Role |
|---|---|
| Notion | Source-of-truth dashboards, databases, tables, views, decisions, tasks, notes |
| GitHub | Versioned front-end code, static data snapshots, documentation, release history |
| Netlify | Hosted interactive interface, visual portal, public/private preview layer |

## Notion should own

- REDBIRD Dashboard
- Roadmap + Tasks
- Current Release Package
- Audio + Listening
- Files + Assets
- Visual + Zine Assets
- Release Calendar
- Decision Log
- Reference Shelf / Back Matter

## GitHub should own

- HTML/CSS/JS interface
- Version notes
- Static JSON data snapshots
- Dashboard schema
- Deployment documentation
- Future lightweight sync scripts if needed

## Netlify should own

- Live preview
- Hosted portal
- Optional password protection later
- Optional serverless functions later, only if static JSON is not enough

## v0.9 scope

Build a useful static dashboard layer first.

This means the interface should include:

- Dashboard overview
- Release readiness indicators
- Timeline / calendar module
- Current release status
- Task/readiness categories
- Listening notes summary
- Files/assets summary
- Decision log summary
- Project data-source map

## v1.0 scope

After the Notion pages are more dashboard-like, create a repeatable export/snapshot process.

Possible data path:

1. Notion dashboards and databases are cleaned.
2. Key values are exported or summarized into JSON.
3. JSON is committed to GitHub under `/data`.
4. Netlify rebuilds or serves the updated static data.

## v1.1 possible scope

Only add live Notion querying if necessary.

That would likely require:

- Netlify Functions
- a Notion integration token stored as an environment variable
- private/public access decisions
- careful filtering so no private source data leaks publicly

## Working rule

Notion remains canonical. GitHub holds interface and snapshots. Netlify displays the usable portal.

Do not make Netlify the source-of-truth. Do not manually maintain the same information in three places unless it is clearly marked as a snapshot.
